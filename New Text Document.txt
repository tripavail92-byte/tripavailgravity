# TripAvail Home Screen - Complete Code Documentation

## Overview
The Home Screen is the main landing page for travelers in the TripAvail application. It features a dynamic, mobile-first design with trending destinations, featured hotels, and featured tours. The screen adapts based on the active bottom navigation tab, showing different content for home, hotels, tours, messages, and profile tabs.

## Component Breakdown

### 1. Main HomeScreen Component

#### Purpose
Acts as the primary routing component that conditionally renders different views based on the active tab.

#### State Management
```typescript
const [selectedPackageId, setSelectedPackageId] = useState<string | null>(null);
```

#### Tab Routing Logic
- **`activeTab === 'hotels'`**: Shows hotel packages or package details
- **`activeTab === 'tours'`**: Shows tour packages placeholder
- **`activeTab === 'messages'`**: Shows messages placeholder
- **`activeTab === 'profile'`**: Shows profile placeholder
- **`showDashboard === true`**: Shows traveler dashboard
- **Default**: Shows trending destinations + featured hotels + featured tours

#### Key Methods
```typescript
// Handle package selection
const handlePackageSelect = (packageId: string) => {
  setSelectedPackageId(packageId);
  onDetailScreenChange?.(true);
};

// Navigate back from package detail

  setSelectedPackageId(null);
  onDetailScreenChange?.(false);
};

// Handle booking action
const handleBookNow = (packageId: string) => {
  console.log('Booking package:', packageId);
  onNavigate('booking');
};

// Toggle wishlist

  wishlistService.toggleWishlist(packageId);
```

#### State
```typescript

```

#### Auto-Slide Implementation
```typescript
useEffect(() => {
  if (images.length <= 1) return; // Skip if single image
  
  const timer = setInterval(() => {
    setCurrentImageIndex((prev) => (prev + 1) % images.length);
  }, autoSlideDelay);

  return () => clearInterval(timer);
}, [images.length, autoSlideDelay]);
```

## Image Optimization

### ImageWithFallback Component
All images use the custom `ImageWithFallback` component for:
- **Graceful Degradation**: Fallback UI if image fails
- **Performance**: Lazy loading built-in
- **Accessibility**: Proper alt text handling

### Unsplash Images
All images are sourced from Unsplash with optimized parameters:
```
?crop=entropy&cs=tinysrgb&fit=max&fm=jpg&ixid=...&ixlib=rb-4.1.0&q=80&w=1080
```

Parameters:
- **crop=entropy**: Smart cropping
- **q=80**: 80% quality for balance
- **w=1080**: Max width 1080px
- **fm=jpg**: JPEG format

## Troubleshooting

### Common Issues

#### 1. Images Not Loading
- **Check**: Unsplash URLs are valid
- **Solution**: Use ImageWithFallback component

#### 2. Wishlist Not Persisting
- **Check**: wishlistService is properly imported
- **Solution**: Verify localStorage is available

#### 3. Animations Stuttering
- **Check**: Too many animations running
- **Solution**: Reduce concurrent animations

#### 4. Slider Not Auto-Advancing
- **Check**: useEffect cleanup

- **Solution**: Use proper theme tokens
